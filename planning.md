### Tool 1: search_listings

**What it does:**
<!-- Describe what this tool does in 1–2 sentences -->
The toolsearches the loaded marketplace dataset to find items matching the user's keywords, size, and price ceiling.

**Input parameters:**
<!-- List each parameter, its type, and what it represents -->
- `description` (str): a couple of keywords about the item
- `size` (str): the closthing size
- `max_price` (float): the maximum amount the user wants to pay

**What it returns:**
<!-- Describe the return value — what fields does a result contain? -->
Retusn a dictionary sorted by relevance and each item in the dictionary contains fields like id, title, condition, etc
**What happens if it fails or returns nothing:**
<!-- What should the agent do if no listings match? -->
It will return an empty list. the agent will kill the loop, drop an error message, and tell the user some hints to fix the search query.

### Tool 2: suggest_outfit

**What it does:**
<!-- Describe what this tool does in 1–2 sentences -->
Takes the top clothing item found in the search and mixes it with the user's existing wardrobe to build an outfit that suits their interests.
**Input parameters:**
<!-- List each parameter, its type, and what it represents -->
- `new_item` (dict): chosen item from the searh step
- `wardrobe` (dict): user's full list of wardrobe items

**What it returns:**
A string containing a detailed outfit recommendation
**What happens if it fails or returns nothing:**
<!-- What should the agent do if the wardrobe is empty or no outfit can be suggested? -->
If the wardrobe is empty or no styling matches make sense, it returns a backup suggestion using generic wardrobe staples

### Tool 3: create_fit_card

new_item: (dict): dictionary of the purchased item to pull the price and platform

**What it returns:**

A string containing the draft caption with emojis and platform details
**What happens if it fails or returns nothing:**

If inputs are missing, it falls back to a generic template caption using just the item title and price

## A Complete Interaction (Step by Step)
The app takes what the user is looking for and tries to find a matching listing first. If a good item pops up, it loops in the user's existing wardrobe to build a styled look and drafts a quick social media caption. If the search comes up empty, the process immediately cuts off and tells the user to try different keywords instead of crashing the outfit generator.

Write out what a full user interaction looks like from start to finish — tool call by tool call. Use a specific example query.

**Example user query:** "I'm looking for a vintage graphic tee under $30. I mostly wear baggy jeans and chunky sneakers. What's out there and how would I style it?"

**Step 1:**
<!-- What does the agent do first? Which tool is called? With what input? -->
Agent extracts variables from the query and calls search_listings("vintage graphic tee", size="M", max_price=30.0). It returns a list of 3 matching tees.
**Step 2:**
<!-- What happens next? What was returned from step 1? What tool is called now? -->
Agent selects the top match (e.g., "Faded Band Tee — $22, Depop"), saves it to the session, and calls suggest_outfit(new_item, wardrobe). It returns a text block telling the user to pair it with their baggy jeans.
**Step 3:**
<!-- Continue until the full interaction is complete -->
Agent takes that styling text and calls create_fit_card(outfit, new_item). It returns a short draft caption with a black heart emoji.
**Final output to user:**
<!-- What does the user actually see at the end? -->
The user sees the available tee found, the advice on how to wear it with their current clothes, and the pre-written caption ready to copy-paste.
